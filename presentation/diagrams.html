Entity Relationship (ER) Diagram
Database schema showing all 6 tables, their attributes, primary/foreign keys, and relationships.
erDiagram
    users {
        int id PK
        varchar name
        varchar email
        varchar student_id
        varchar password_hash
        enum role
        boolean is_active
        boolean must_change_password
        datetime created_at
    }

    menus {
        int id PK
        date date
        enum meal_type
        text description
        datetime created_at
    }

    menu_items {
        int id PK
        int menu_id FK
        varchar name
        varchar description
    }

    attendance_confirmations {
        int id PK
        int user_id FK
        int menu_id FK
        datetime confirmed_at
        boolean attended
    }

    notifications {
        int id PK
        int user_id FK
        varchar title
        text message
        boolean is_read
        datetime created_at
    }

    password_reset_codes {
        int id PK
        int user_id FK
        varchar code
        datetime expires_at
        boolean used
        datetime created_at
    }

    users ||--o{ attendance_confirmations : "confirms meal"
    menus  ||--o{ attendance_confirmations : "confirmed by"
    menus  ||--o{ menu_items              : "has items"
    users  ||--o{ notifications           : "receives"
    users  ||--o{ password_reset_codes    : "has reset code"
PK — Primary Key
FK — Foreign Key
||--o{ — One to Many relationship
role ENUM — student / staff / admin
meal_type ENUM — breakfast / lunch / dinner

System Flowcharts
Step-by-step process flows for the two core workflows: user authentication and meal attendance confirmation.
Login / Authentication Flow
flowchart TD
    A([User visits /]) --> B{Authenticated?}
    B -- Yes --> C{Role?}
    B -- No --> D[/auth/login page]
    C -- admin or staff --> E[/admin/dashboard]
    C -- student --> F[/student/dashboard]
    D --> G[Enter email + password]
    G --> H[POST /auth/login]
    H --> I{User found & active?}
    I -- No --> J[Flash: Invalid credentials]
    J --> D
    I -- Yes --> K{must_change\n_password?}
    K -- Yes --> L[/auth/change-password]
    L --> M[Submit new password]
    M --> N{Passes strength\nvalidation?}
    N -- No --> O[Flash error] --> L
    N -- Yes --> P[Save new password hash\nSet flag = False]
    P --> C
    K -- No --> C

Meal Attendance Confirmation Flow
flowchart TD
    A([Student views /student/dashboard]) --> B[Server loads today's menus\n+ student's confirmed IDs]
    B --> C[Render meal cards\nwith Confirm/Cancel buttons]
    C --> D[Student clicks button]
    D --> E[JS: show spinner,\ndisable button]
    E --> F[fetch POST /student/confirm/menu_id]
    F --> G{Is meal date\nin the past?}
    G -- Yes --> H[Return error JSON]
    H --> I[Show danger toast\nRestore button]
    G -- No --> J{Already confirmed?}
    J -- Yes --> K[DELETE confirmation record]
    K --> L[Return: status=cancelled]
    J -- No --> M[INSERT confirmation record]
    M --> N[Return: status=confirmed]
    L --> O[Update card UI: remove\nborder, badge, toggle button]
    N --> P[Update card UI: add\ngreen border, badge, toggle button]
    O --> Q[Show info toast]
    P --> R[Show success toast]

Menu Creation Flow (Admin / Staff)
flowchart LR
    A([Admin/Staff logs in]) --> B[Navigate to Add Menu\nor Weekly Planner]
    B --> C{Single menu\nor weekly?}
    C -- Single --> D[Fill date, meal type,\ndescription, food items]
    C -- Weekly --> E[Fill 7-day × 3-meal grid\nwith descriptions + items]
    D --> F[POST /admin/menus/add]
    E --> G[POST /admin/menus/weekly]
    F --> H{Date + meal_type\nalready exists?}
    H -- Yes --> I[Flash: duplicate error\nRedirect back to form]
    H -- No --> J[INSERT Menu record\nINSERT MenuItem rows]
    G --> K[For each filled slot:\nCheck for duplicates\nSkip existing, create new]
    J --> L[Flash success\nRedirect to /admin/menus]
    K --> L
